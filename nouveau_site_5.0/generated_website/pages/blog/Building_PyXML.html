Building My Own PyXML Compiler
<h3 id="why-pyxml">Why PyXML?</h3> <p>Standard SSGs like Jekyll or Hugo use Markdown + YAML Frontmatter. While powerful, they often feel disconnected from the HTML structure. PyXML allows me to define <strong>custom semantic tags</strong> that map directly to high-level UI components.</p> <h3 id="implementation-details">Implementation Details</h3> <ul> <li><strong>Parser</strong>: Uses Python's <code>xml.etree.ElementTree</code> with custom extensions for execution blocks (<code>exec_*</code>).</li> <li><strong>Context</strong>: A robust state manager for variable resolution and pathing.</li> <li><strong>Glassmorphism</strong>: Native support for CSS backdrop filters.</li> </ul> <h3 id="future-improvements">Future Improvements</h3> <ul> <li><strong>Live Preview</strong>: Integrated dev server with hot reload.</li> <li><strong>Wasm Version</strong>: Running the compiler in the browser.</li> </ul>
<h3 id="pourquoi-pyxml">Pourquoi PyXML ?</h3> <p>Les générateurs de sites statiques standards comme Jekyll ou Hugo utilisent Markdown + YAML Frontmatter. Bien que puissants, ils donnent souvent l'impression d'être déconnectés de la structure HTML. PyXML me permet de définir des <strong>balises sémantiques personnalisées</strong> qui correspondent directement à des composants d'interface utilisateur de haut niveau.</p> <h3 id="details-dimplementation">Détails d'implémentation</h3> <ul> <li><strong>Parser</strong>: Utilise <code>xml.etree.ElementTree</code> de Python avec des extensions personnalisées pour les blocs d'exécution (<code>exec_*</code>).</li> <li><strong>Contexte</strong>: Un gestionnaire d'état robuste pour la résolution des variables et la gestion des chemins.</li> <li><strong>Glassmorphism</strong>: Support natif pour les filtres d'arrière-plan CSS.</li> </ul> <h3 id="ameliorations-futures">Améliorations futures</h3> <ul> <li><strong>Aperçu en direct</strong>: Serveur de développement intégré avec rechargement à chaud.</li> <li><strong>Version Wasm</strong>: Exécution du compilateur dans le navigateur.</li> </ul>
